DESCRIPCIÓN:
Mantenimiento de una tabla. Permite inserción, actualización y borrado de los datos correspondientes al diccionario. El funcionamiento de todos los diccionarios es similar, por lo que este Caso de Uso es válido para la mayoría de ellos.



PRECONDICIÓN:
El usuario está autenticado y tiene permisos para acceder aquí. Es decir cumple el CU-AD0005-Seguridad.

SECUENCIA NORMAL:
1.-El usuario selecciona la opción de Mantenimiento del diccionario X.
2.-El sistema comprueba que tiene permiso y presenta una pantalla para introducir un filtro. Activa la opcion de insertar solo si el usuario tiene el permiso correspondiente asociado.
3.-El usuario introduce el filtro y pulsa la opción de buscar.
4.-El sistema presenta una lista de datos que cumplen con el filtro.
5.-El usuario selecciona el registro que desea modificar o ver.
6.-El sistema presenta los datos del registro seleccionado recogiéndolos de la base de datos. Pone todos los campos de solo lectua. Activa las opciones de modificar y borrar solo si el usuario tiene los permisos adecuados.
7.-El usuario selecciona la opcion de modificar.
8.-El sistema pone los campos modificables.
9.-El usuario modifica los datos y solicita al sistema que guarde los cambios.
10.-El sistema comprueba que los datos son válidos (son obligatorios los definidos en la bb.dd y la longitud máxima no puede superar lo definido en la misma) y almacena los cambios. Actualiza igualmente los campos de auditoría de modificación de datos.


SECUENCIAS ALTERNATIVAS:
2.-Si el usuario no tiene permiso, el sistema presenta un mensaje indicándolo.

3.-Si el usuario selecciona la opción de insertar, se ejecuta el CU-AD0032.

7.-Si el usuario selecciona la opción de eliminar registro, se ejecuta el CU-AD0034.

10.- Si los datos introducidos no son válidos, presentar mensaje de error.

3 o 7-Si el usuario selecciona la opción "Navegar", se ejecuta el caso de uso CU-AD0060

PLAN DE PRUEBAS:
- Los registros de la lista todos los que cumplen el filtro y sólo los que cumplen el filtro.
- Al actualizar un registro se modifican los datos de auditoría con el usuario que ha realizado la modificación y se mantienen los datos de creación el registro.
- Realizar las pruebas definidas en la checklist del documento de instrucción técnica.
