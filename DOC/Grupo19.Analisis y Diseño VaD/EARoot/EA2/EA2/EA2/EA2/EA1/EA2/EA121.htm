UC. Habilitado para perfil admiistrador. No incluido en esta versión VaD_v1.0.
Actores Usuario Admin
Propósito Dar de alta un nuevo perfil
Resumen El usuario accede a la ventana de mantenimiento de perfiles y solicita dar de alta uno nuevo, mostrándose la ventana en la que el usuario introduce los valores del nuevo perfil.
Tipo Primario y Esencial
Referencias Cruzadas Casos de uso: el usuario debe haber terminado el caso de uso Acceso al Sistema (UC-26) y Consulta Perfil (UC-33).
Requisitos Asociados
Curso normal de los eventos
Acciones de los actores Respuesta del Sistema
1. El Usuario accede a la ventana de mantenimiento de perfiles (AD-2050).
2. Muestra la ventana del mantenimiento de perfiles con todos los perfiles ya existentes, sus funcionalidades y sus acciones.
3. El Usuario solicita dar de alta un nuevo perfil.
4. Muestra la ventana para introducir los valores del nuevo registro.
5. El Usuario introduce los datos del nuevo perfil y solicita almacenarlo.
6. Solicita confirmación del alta.
7. El Usuario confirma el alta del nuevo perfil.
8. Graba los datos del nuevo registro y muestra una ventana con los datos de los perfiles almacenados, sus funcionalidades y acciones.
Cursos alternos
Línea 2: El Usuario no tiene permisos para realizar el alta de perfiles. El Sistema muestra mensaje informando y cancela el proceso.
Línea 7: El Usuario cancela el alta. El Sistema cancela el proceso
Línea 8: El Usuario no introduce todos los datos obligatorios del perfil a dar de alta. El Sistema muestra mensaje informando al respecto y solicita la inserción de todos los datos obligatorios antes de almacenar el nuevo perfil, no permitiendo su inserción hasta que todos esos datos estén cumplimentados.
