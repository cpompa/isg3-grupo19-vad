DESCRIPCIÓN:
Proceso de autenticación de usuario y construcción del menú según los permisos que le corresponda.


PRECONDICIÓN:
El usuario no está autenticado.


SECUENCIA NORMAL:
1.-El usuario intenta acceder a cualquier página del sistema.
2.-El sistema pide login y contraseña.
3.-El usuario introduce su login y contraseña.
4.-El sistema comprueba que el usuario y contraseña corresponde a un usuario registrado y que es correcto, y las opciones de menú a las que tiene acceso. Construye el menú personalizado y se lo presenta al usuario.


SECUENCIAS ALTERNATIVAS:
4.-Si el usuario no ha introducido un par usuario-clave correcto, se vuelve a pedir.


POSTCONDICIÓN:
El usuario está logado y con el menú construido con todas las opciones que tiene disponible.


REFERENCIAS:
ERD-AD0010
DC-AD0010


PLAN DE PRUEBAS:
* Insertar los datos necesarios en la base de datos para disponer de un usuario con 2 perfiles que tienen alguna opción común.
* Auntenticarse con contraseña equivocada y comprobar que no entra.
* Auntenticarse con la clave correcta y comprobar que las opciones posibles están bien
* Quitar uno de los perfiles y comprobar que las opciones que aparecen son solo las que les corresponde al perfil que queda asignado al usuario
