DESCRIPCIÓN:
Borra un registro de una tabla. Este CU extiende al CU-Mantenimiento.


PRECONDICIÓN: El registro que se pretende borrar existe en la base de datos. El usuario ha seleccionado la opción para eliminar el registro.


SECUENCIA NORMAL:
1.-El sistema comprueba que el usuario tiene permisos y que el registro que se quiere eliminar cumple todos los requisitos para que se pueda borrar. El sistema elimina el registro


SECUENCIA ALTERNATIVA:
1.-Si el usuario no tiene permiso, el sistema presenta un mensaje indicándolo.

1.-Si el sistema detecta que el registro no se puede borrar por cualquier causa, se presenta un mensaje de error indicándolo.


POSTCONDICIÓN: El registro ha sido borrado

REFERENCIAS:

PLAN DE PRUEBAS:
*Crear un registro, vincularlo con otro en caso de que se pueda y eliminarlo, el sistema muestra un error indicando que existen registros relacionados.
*Borrar la relación y borrar el registro. El sistema borra el registro.
