DESCRIPCIÓN:
Consulta de una tabla. Permite la consulta de los datos correspondientes a la tabla. El funcionamiento de todas las consultas es similar, por lo que este Caso de Uso es válido para la mayoría de ellos.



PRECONDICIÓN:
El usuario está autenticado y tiene permisos para acceder aquí. Es decir cumple el CU-AD0005-Seguridad.

SECUENCIA NORMAL:
1.-El usuario selecciona la opción de Consulta de la tabla X.
2.-El sistema comprueba que tiene permiso y presenta una pantalla para introducir un filtro.
3.-El usuario introduce el filtro y pulsa la opción de filtrar.
4.-El sistema presenta una lista de datos que cumplen con el filtro. Esta lista presenta los registros paginados de 10 en 10.
5.-El usuario selecciona el registro que desea consultar.
6.-El sistema presenta los datos del registro seleccionado recogiéndolos de la base de datos.


SECUENCIAS ALTERNATIVAS:
2.-Si el usuario no tiene permiso, el sistema presenta un mensaje indicándolo.


POSTCONDICIÓN:


REFERENCIAS:
CU-AD0005
ERD-AD0010


PLAN DE PRUEBAS:
- Los registros de la lista todos los que cumplen el filtro y sólo los que cumplen el filtro.
- Al consultar un registro se muestran los datos que realmente están en la BBDD.
- Realizar las pruebas definidas en la checklist del documento de instrucción técnica.
