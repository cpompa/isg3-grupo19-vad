DESCRIPCIÓN:
Inserta un registro en una tabla. Este CU extiende al CU-Mantenimiento.


PRECONDICIÓN: El usuario ha seleccionado la opción para añadir registro.


SECUENCIA NORMAL:
1.- El sistema comprueba que el usuario tiene permiso para insertar el registro y presenta el formulario de registro de datos con los valores por defecto si los tiene. Los campos tipo select tendrán un valor vacío que podrá ser seleccionado por el usuario.
2.- El usuario introduce los datos y pulsa la opción para guardar.
3.- El sistema comprueba que los datos son válidos y los almacena. Actualiza igualmente los campos de auditoría de creación de datos.


SECUENCIAS ALTERNATIVAS:
1.- Si el usuario no tiene permiso para insertar registros, se presenta un mensaje indicándolo.

3.- Si los datos introducidos no son correctos, se presenta un mensaje indicándolo.

3.- La clave primaria del registro está duplicada, se presenta un mensaje indicándolo.


POSTCONDICIÓN: El registro ha sido creado




PLAN DE PRUEBAS:
*No informar alguno de los datos obligatorios, el sistema presenta un mensaje de error indicándolo por cada uno de los datos obligatorios.
*Introducir todos los datos obligatorios.
Tanto en la auditoria de creación como de actualización aparecen el usuario que ha realizado la actualización y la fecha del sistema.
Si la tabla a mantener es un maestro-detalle también se ha informado los datos de la relación.
Si la clave primaria de la tabla es informada por el usuario, el error de clave duplicada se controla y se le informa al usuario permitiéndole modificar la clave primaria.
