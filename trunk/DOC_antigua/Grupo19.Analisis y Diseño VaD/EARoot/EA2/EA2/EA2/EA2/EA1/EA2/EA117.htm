Actores Usuario Administrador
Propósito Dar de alta un nuevo Usuario
Resumen El usuario accede a la ventana de mantenimiento de usuarios y solicita dar de alta uno nuevo, mostrándose la ventana en la que el usuario introduce los valores del nuevo usuario.
Tipo Primario y Esencial
Referencias Cruzadas Casos de uso: el usuario debe haber terminado el caso de uso Acceso al Sistema (UC-24) y Consulta Usuario (UC-27).
Requisitos Asociados
Curso normal de los eventos
Acciones de los actores Respuesta del Sistema
1. El Usuario accede a la ventana de mantenimiento de usuarios (CAU-27).
2. Muestra la ventana del mantenimiento de usuarios con todos los usuarios ya existentes.
3. El Usuario solicita dar de alta un nuevo usuario.
4. Muestra la ventana para introducir los valores del nuevo registro.
5. El Usuario introduce los datos del nuevo usuario y solicita almacenarlo.
6. Solicita confirmación del alta.
7. El Usuario confirma el alta del nuevo usuario.
8. Graba los datos del nuevo registro y muestra una ventana con los datos de los usuarios almacenados.
Cursos alternos
Línea 2: El Usuario no tiene permisos para realizar el alta de usuarios. El Sistema muestra mensaje informando y cancela el proceso.
Línea 7: El Usuario cancela el alta. El Sistema cancela el proceso
Línea 8: El Usuario no introduce todos los datos obligatorios del usuario a dar de alta. El Sistema muestra mensaje informando al respecto y solicita la inserción de todos los datos obligatorios antes de almacenar el nuevo usuario, no permitiendo su inserción hasta que todos esos datos estén cumplimentados.
