Item Type Meaning
Arquitectura de Despliegue Technical Es una vista del hardware propuesto que configurará el nuevo sistema, junto con los componentes físicos que se ejecutarán en ese hardware. Incluye las especificaciones de las diferentes máquinas, sistemas operativos, enlaces de red, unidades de backup, etc.
Asociación Technical Es una relación entre dos o más entidades. Implica una conexión del algún tipo - por ejemplo una entidad usa los servicios de otra, o una entidad está conectada con otra a través de un enlace de red.
Caso de Uso Technical Un Caso de Uso representa una unidad discreta de interacción o funcionalidad entre un usuario (persona o sistema) y le propio sistema. Un caso de uso es una unidad sencilla de trabajo significatibo; por ejemplo crear una ficha, modificar un pedido o emitir una factura constituyen ejemplos de casos de uso. Cada Caso de Uso tiene una descripción que detalla la funcionalidad que proporcionará al sistema. Un Caso de Uso puede 'incluir' otros Casos de Uso o 'extender' otro Caso de Uso dentro de su propio comportamiento. Los Casos de Uso están normalmente relacionados con Actores.
Clase Technical Una entidad lógica que encapsula datos y comportamiento. Una clase es una plantilla para un objeto - la clase es un concepto de diseño, y el objeto es la instancia en tiempo de ejecución.
Estrategia de desarrollo Technical Es el enfoque a utilizar para establecer cómo debe organizarse el proyecto.
EVS Technical Estudio de Viabilidad del Sistema. Es uno de los procesos de Métrica 3. Es equivalente al ARS de Métrica 2 y sus productos fundamentales son: Descripción del Ámbito y Alcance del Proyecto, Catálogo de Requerimientos, Solución de Desarrollo Seleccionada, Plan inicial de Trabajo. Se pueden ver las actividades fundamentales de este proceso observando la estructura de proyecto en Enterprise Architect.
Métrica 3 Technical Es la metodología que se emplea para el desarrollo de aplicaciones. Métrica 3 es la metodología de desarrollo de Sistemas de Información especialmente dirigido a las Administraciones Públicas. Incluye un modelo de referencia para desarrollos estructurados y desarrollos orientados a objeto.
Modelo de Componentes Technical El modelo de componentes proporciona una visión detallada de los diferentes componentes software y hardware que configuran el sistema propuesto. Muestra cómo interactúan estos componentes y el lugar donde residen. Los requerimientos de los componentes detallan que responsabilidades u operaciones debe proporcionar dentro del sistema.
Modelo de Despliegue Technical Un modelo del sistema tal como se desplagará físicamente.
Paquete Technical Artefacto UML que agrupa elementos de la misma naturaleza. Es de propósito general, y eromite agrupar tanto elementos del proceso (actividades) como elementos del Proyecto (modelos y diagramas)
Proyecto Business Un proyecto es una petición de un departamento aceptada para desarrollar una solución informática
