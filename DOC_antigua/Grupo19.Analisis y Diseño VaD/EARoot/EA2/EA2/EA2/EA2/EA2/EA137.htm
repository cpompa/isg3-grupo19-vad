Actores Usuario Registrado como Demandante
Propósito Modificar una Solicitud
Resumen El Usuario modifica los datos de alguna de las solicitudes asociadas.
Tipo Primario y Esencial
Referencias Cruzadas Casos de uso: el usuario debe haber terminado los casos de uso Acceso al Sistema (UC-24) y Consultar Solicitud (UC-20).
Requisitos Asociados
Curso normal de los eventos
Acciones de los actores Respuesta del Sistema
1.a El Usuario accede a la opción de menu de inicio y solicita ver las solicitudes (UC-20).
2. Muestra la ventana con los datos de las solicitudes activas.
3. El Usuario selecciona una solicitud, modifica sus datos y solicita modificarla.
4. Solicita confirmación de la operación.
5. El Usuario confirma la modificación de la solicitud.
6. Modifica la solicitud.
Cursos alternos
Línea 1: El Usuario no tiene permisos para modificar solicitudes. El Sistema muestra mensaje informando y cancela el proceso.
Línea 3: No existen solicitudes.
Línea 5: El Usuario cancela la asociación. El Sistema cancela el proceso.
