DESCRIPCIÓN:
Proceso de control del acceso del usuario a las páginas accedidas mediante su perfil


PRECONDICIÓN:


SECUENCIA NORMAL:
1.-El usuario intenta acceder a cualquier página restringida.del sistema.
2.-El sistema comprueba que el usuario tiene permiso para acceder a la página.
3.-El usuario accede a la página.


SECUENCIAS ALTERNATIVAS:
2.-Si el usuario no está logado, el sistema redirige a la página de login. Iniciándose así el caso de uso CU-AD0010
2.-Si el usuario está logado pero no tiene permiso para acceder a la página, el sistema redirige a la página de error indicando al usuario que no tiene permiso para acceder a la página.


POSTCONDICIÓN:


REFERENCIAS:
CU-AD0010



PLAN DE PRUEBAS:
*Crear usuario sin perfiles.
*Logarse y pedir otra vez la página, el sistema muestra la página de error indicando que el usuario no tiene permiso para acceder a la página.
*Modificar al usuario añadiéndole todos los perfiles.
En otro navegador pedir la página misma página, el sistema redirige a la página de Login.
*Logarse y pedir otra vez la página, el sistema muestra la página solicitada.
